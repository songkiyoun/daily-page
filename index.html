상단 + 일정 버튼 → 날짜·시간·장소 직접 입력. 등록된 일정 카드 클릭 시 수정, ✕ 버튼으로 삭제 가능.
🔄 동기화 설정 (최초 1회)
1
구글 시트 만들기
sheets.new → 빈 시트 생성 → 이름 지정 (예: 일정)
2
Apps Script 열기
시트 상단 메뉴 확장 프로그램 → Apps Script 클릭
3
코드 붙여넣기
편집창 기존 코드 전체 삭제 → AppScript.gs 내용 전체 붙여넣기 → Ctrl+S 저장
4
웹 앱으로 배포
상단 배포 → 새 배포
· 유형: 웹 앱
· 다음 사용자로 실행: 나
· 액세스 권한: 모든 사용자
→ 배포 → 권한 승인 → URL 복사
액세스 권한을 반드시 모든 사용자로 설정해야 합니다.
5
앱에 URL 연결
상단 🔗 URL 변경 → 복사한 URL 붙여넣기 → 연결. 모바일에서도 동일한 URL 입력하면 데이터 공유 시작.
https://script.google.com/macros/s/AKfy.../exec
데이터는 시트 A1(일정), B1(할일) 셀에 JSON으로 자동 저장됩니다. 변경 후 약 1.5초 뒤 자동 저장됩니다.
💡 동기화 상태 표시
●
저장 완료 / 동기화 완료
데이터가 정상적으로 시트에 저장됐습니다.
●
저장 중 / 동기화 중
데이터를 시트에 저장하는 중입니다.
●
오류
↻ 동기화 버튼으로 재시도. 반복 시 URL 재설정이 필요할 수 있습니다.
❓ 자주 묻는 질문
모바일에서 앱처럼 사용하려면?
GitHub Pages로 배포 후 Safari → 공유 → 홈 화면에 추가 (아이폰), Chrome → 메뉴 → 홈 화면에 추가 (안드로이드)
동기화가 계속 실패하면?
Apps Script에서 배포 → 기존 배포 관리 → 새 버전으로 다시 배포한 후 새 URL을 🔗 URL 변경으로 다시 입력해주세요.
데이터가 안전한가요?
Apps Script URL을 아는 사람만 접근 가능합니다. URL을 외부에 공유하지 않는 한 안전하며, 데이터는 본인 Google 계정 시트에 저장됩니다.
유튜브/클라이언트 데이터가 안 불러와지면?
프록시 서버 상태에 따라 간헐적으로 실패할 수 있습니다. ↻ 새로고침을 눌러 재시도하세요.
