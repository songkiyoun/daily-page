# Circle Battle Tower Rebuild

## v0.7.4 핵심 변경

- 1층 상대 스탯을 힘·체력·방어·민첩·행운 모두 5로 고정했습니다.
- 1층 상대는 숙련도 0, 스킬 없음 상태로 시작해 무기·성격 테스트 기준이 더 명확해졌습니다.
- 전투 시뮬레이터를 추가했습니다. 현재 플레이어 무기·성격과 선택한 상대 조합을 1층 기본 스탯 5 기준으로 반복 테스트할 수 있습니다.
- 시뮬레이터는 선택 조합 테스트와 상대 전체 비교를 지원합니다.
- 시뮬레이션은 실제 전투 로직을 사용하되 화면 렌더링 없이 빠르게 반복합니다.

## 이번 버전에서 중점적으로 볼 것

1. 1층 상대 스탯이 전부 5로 표시되는지 확인합니다.
2. 같은 조합을 여러 번 시뮬레이션했을 때 승률·평균 명중·평균 시간이 조합별 차이를 보여주는지 확인합니다.
3. 실제 전투 체감과 시뮬레이터 결과가 지나치게 다르지 않은지 비교합니다.
4. 밸런스 판단은 1회 전투 결과보다 30~50회 이상 평균 결과를 기준으로 봅니다.

## v0.6.3 핵심 변경

- 성격 시스템을 단순 AI 성향에서 전투 수치 보정 구조로 재정의했습니다.
- 공격형은 공격력, 공격속도, 밀쳐내기, 자세 압박이 강해지고 방어·패링 안정성이 낮아집니다.
- 방어형은 공격력과 속도가 낮아지는 대신 방어력, 패링, 밀림 저항, 자세 안정성이 높아집니다.
- 밸런스형은 모든 보정이 기준값으로 유지되어 테스트 기준 성격 역할을 합니다.
- 암살형은 이동, 회전, 측후방 진입, 치명타가 좋아지는 대신 방어력과 밀림 저항이 낮아집니다.
- 무기 정체성은 유지하고, 성격은 그 무기를 운용하는 방식과 수치를 보정하도록 정리했습니다.

## 이번 버전에서 중점적으로 볼 것

1. 공격형이 단순 돌진이 아니라 실제 공격 속도와 압박이 좋아지는 대신, 패링·방어에서 약점이 생기는지 확인합니다.
2. 방어형이 오래 버티고 덜 밀리지만 공격이 답답해지는지 확인합니다.
3. 암살형이 빨라지고 측후방 진입은 좋아지지만 맞았을 때 더 위험해지는지 확인합니다.
4. 같은 무기라도 성격에 따라 전투 결과가 달라지되, 무기 자체의 정체성이 깨지지 않는지 확인합니다.

## v0.6.0 핵심 변경

- 민첩 보정이 모든 무기를 비슷하게 빠르게 만들지 않도록, 무기별 민첩 반영 비율과 상한을 분리했습니다.

## 핵심 방향

- 1:1 전투 전용
- 2:2, 3:3, 아군, 팀전 구조 제외
- 사각형 전장
- 매 층 랜덤 상대 생성
- 층이 오를수록 상대 체력, 공격, 방어, 스킬 구성 상승
- 플레이어 스탯, 경험치, 레벨, 무기 숙련, 기본 스킬 구조 추가

- 새 런 시작 시 기본 스탯 포인트 제공
- 무기별 방향 전환 속도, 공격 후딜, 자세 게이지, 자세 흐트러짐 반영

- 초기 중앙 전투 버튼이 새 런을 다시 만들며 스탯을 초기화하던 문제 수정
- 창·방어형 미러전에서 뒤로만 거리두는 루프 완화
- 원거리에서 밀치기·무기 충돌로 자세가 흐트러지던 오판정 완화
- 단검 미러전에서 붙어서 빙글 도는 루프를 짧은 교전으로 전환
- 무기 충돌과 공격 성공 시 상대의 몸 방향이 즉각 흐트러지도록 반영
- 캐릭터 세팅, 중앙 전투·결과·보상, 우측 탑 정보·상태 UI 구조 적용
- 클릭 가능한 UI는 매 프레임 재생성하지 않고 상태 변화 시에만 갱신

- 패링 기본형, 무기별 패링 효율, 반격 창 추가
- 무기별 공격 준비·타격·후딜 리듬 강화
- 무기별 공격 진입 동작을 창 찌르기, 서양검 전진 베기, 동양검 사선 진입, 단검 페이크 찌르기로 분리
- 명중, 패링, 무기 충돌 반응을 창 큰 넉백, 서양검 자세 붕괴, 동양검 짧은 압박, 단검 치고 빠지기로 차별화
- 명중, 패링, 무기 충돌 시 짧은 충격 정지로 타격 순간 강조
- 핵심 이벤트만 캔버스 텍스트로 표시해 과한 텍스트 난립을 방지
- 단검 페이크, 반대 꺾기, 순간 침투, 명중 후 재정렬 구조 적용
- 공격 상태에 따라 무기 궤적과 타격 범위가 더 분명하게 표시
- 패배하면 새 런 시작

## 파일 구조

```text
circle-battle-rebuild/
├─ index.html
├─ css/
│  └─ style.css
├─ js/
│  ├─ main.js
│  ├─ data.js
│  ├─ state.js
│  ├─ battle.js
│  ├─ ai.js
│  ├─ render.js
│  └─ utils.js
├─ README.md
├─ CHANGELOG.md
├─ DEVELOPMENT_RULES.md
├─ .gitignore
└─ .nojekyll
```

## 파일별 역할

- `index.html`: 화면 구조
- `css/style.css`: 디자인
- `js/data.js`: 무기, 성격, 스탯, 스킬, 보상, 층 성장 규칙
- `js/state.js`: 런 생성, 층 이동, 유닛 생성, 랜덤 상대 생성, 보상 적용
- `js/battle.js`: 공격 판정, 피해 계산, 승패 판정, 사각형 경계 처리
- `js/ai.js`: 1:1 자동 이동 판단
- `js/render.js`: 캔버스 그리기
- `js/main.js`: UI 연결과 단일 게임 루프
- `js/utils.js`: 공통 수학 함수

## 로컬 실행

`type="module"`을 사용하므로 파일을 더블클릭하면 브라우저 보안 정책에 막힐 수 있습니다.
아래처럼 로컬 서버로 실행하세요.

```bash
python -m http.server 8000
```

브라우저에서 접속합니다.

```text
http://localhost:8000
```

## 개발 원칙

- 기존 파일 아래에 패치 블록을 추가하지 않습니다.
- 같은 이름의 함수를 다시 선언하지 않습니다.

- 동양검 상대로 단검이 과하게 유리해지는 문제를 줄이기 위해 후딜 자동 진입을 제한했습니다.


## v0.6.15 단검 측면·후면 공격 우선순위 강화

이번 버전은 v0.6.14의 정면 유인형 단검 구조를 유지하면서, 측면·후면을 이미 잡은 상황에서 유인/회피 모드로 되돌아가는 문제를 줄인 버전입니다.

- 단검이 측면·후면 각도를 잡은 상태에서는 정면 유인이나 사선 회피보다 공격 대기/진입을 우선합니다.
- 창의 후면을 잡았는데도 공격하지 않고 거리만 유지하던 문제를 완화했습니다.
- 창 상대로 측면·후면 진입 허용폭을 넓히되, 단검이 창을 압도하지 않도록 대창 피해/자세 보상을 낮췄습니다.
- 공격 쿨타임이 조금 남아 있을 때는 뒤로 빠지는 대신 측면·후면 공격 대기를 유지합니다.
- 서양검·동양검 상대로는 v0.6.14의 유인형 구조를 대부분 유지했습니다.

## v0.7.1 스킬 발동 문구 및 모션 보정

이번 버전은 v0.7.0의 스킬 뼈대에서 확인된 표시/발동 문제를 보정한 버전입니다.

- 서양검 베쉬 발동 문구를 `베쉬!`로 표시하도록 수정했습니다.
- 단검 급소 찌르기 발동 문구를 `급소 찌르기!`로 표시하도록 수정했습니다.
- 단검 급소 찌르기가 측면/측후면/후면 상황에서 더 안정적으로 발동되도록 각도 판정을 완화했습니다.
- 방어형 피해반사의 발동 조건을 강화하고 쿨타임을 길게 늘렸습니다.
- 창 연속 찌르기가 단순 추가 피해가 아니라 짧은 추가 공격 상태로 이어지도록 수정했습니다.
- 창 벤다!가 단순 문구가 아니라 창 휘두르기 공격 상태로 이어지도록 수정했습니다.
- 단검 분신 인형 문구를 `분신 인형!`으로 표시하고, 큰 피해 또는 위험 피해 상황에서 더 잘 발동되도록 수정했습니다.
- 서양검 베쉬, 창 연속 찌르기, 창 벤다!에 스킬 전용 공격 라벨과 일부 모션 보정을 추가했습니다.


## v0.7.2 창 벤다! 반격 구조 수정

이번 버전은 창의 `벤다!` 스킬을 근접 자동 발동기에서 측후면 반격기로 바꾼 버전입니다.

- 기존 `벤다!`의 단순 근접 자동 발동을 제거했습니다.
- 창이 측면, 측후면, 후면에서 공격받을 때 확률적으로 `벤다!`가 발동되도록 수정했습니다.
- `벤다!` 발동 시 창 사용자가 공격자 방향으로 즉시 방향 전환합니다.
- `벤다!` 발동 시 공격자를 밀어내고, 창 휘두르기 공격 상태로 진입합니다.
- 렌더링에서도 `spearSweep` 전용 휘두르기 각도와 넓은 호를 사용하도록 보정했습니다.
- `집중`은 방어적 방향 전환과 약한 밀어내기, `벤다!`는 반격 휘두르기로 역할을 분리했습니다.


## v0.7.3 창 집중 역할 변경

이번 버전은 창의 `집중` 스킬을 측후면 방어 반응에서 자세 복구형 안정화 스킬로 바꾼 버전입니다.

- `벤다!`는 측면, 측후면, 후면 공격을 받았을 때 나오는 반격 베기로 유지했습니다.
- `집중`은 더 이상 측후면 공격 반응으로 발동하지 않습니다.
- `집중`은 창의 자세 게이지가 낮아졌을 때 자동 발동합니다.
- `집중` 발동 시 자세 게이지를 일부 회복합니다.
- `집중` 발동 시 상대 방향으로 즉시 방향을 보정합니다.
- `집중` 발동 후 짧은 시간 동안 창의 방향 전환 속도가 증가합니다.
- `집중` 발동 후 다음 찌르기 준비 시간이 소폭 줄어듭니다.

### 창 스킬 역할 분리

- 연속 찌르기: 공격 성공 후 추가 압박
- 벤다!: 측후면 침투 대응 반격
- 집중: 자세 붕괴 대응 복구
