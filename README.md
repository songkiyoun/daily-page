# Circle Battle Tower Rebuild

## v0.5.0 핵심 변경

- 무기별 공격 준비, 타격, 후딜 리듬을 강화했습니다.
- 창은 긴 준비와 강한 찌르기, 긴 회수 동작이 더 분명해졌습니다.
- 서양검은 크게 감고 넓게 베며, 헛쳤을 때 빈틈이 더 커졌습니다.
- 동양검은 짧은 준비와 빠른 베기, 빠른 이탈 리듬을 유지합니다.
- 단검은 짧은 찌르기와 빠른 이탈을 살리되, 측후방 기회를 노리는 구조를 유지합니다.
- 공격이 명중하거나 패링될 때 짧은 충격 정지가 발생해 타격 순간이 더 잘 보입니다.
- 캔버스에서 무기 궤적과 타격 범위가 공격 상태에 따라 더 분명하게 표시됩니다.

## v0.4.9 핵심 변경

- 정면 공격을 받아낼 수 있는 패링 기본형을 추가했습니다.
- 무기별 패링 효율과 성격별 패링 보정이 적용됩니다.
- 패링 성공 시 공격자의 자세가 크게 깎이고 몸 방향이 틀어지며 짧은 후딜이 발생합니다.
- 패링 성공자는 짧은 반격 창을 얻어 다음 공격의 체력 피해와 자세 피해가 소폭 증가합니다.
- 패링 성공은 캔버스에서 짧은 푸른 링으로 표시됩니다.
- 모든 수정은 기존 담당 파일의 기존 함수와 데이터 구조를 직접 수정하는 방식으로 반영했습니다.



1:1 자동 결투 기반 로그라이크 탑 등반 게임입니다.
기존 실험본의 패치 누적 구조를 버리고, GitHub Pages에 올릴 수 있는 분리형 프로젝트로 다시 구성했습니다.

## 현재 버전

v0.5.0

## 핵심 방향

- 1:1 전투 전용
- 2:2, 3:3, 아군, 팀전 구조 제외
- 사각형 전장
- 매 층 랜덤 상대 생성
- 층이 오를수록 상대 체력, 공격, 방어, 스킬 구성 상승
- 플레이어 스탯, 경험치, 레벨, 무기 숙련, 기본 스킬 구조 추가
- 승리하면 보상 선택 후 체력을 완전히 회복하고 다음 층으로 이동
- 새 런 시작 시 기본 스탯 포인트 제공
- 무기별 방향 전환 속도, 공격 후딜, 자세 게이지, 자세 흐트러짐 반영
- 방어형과 창의 무한 후퇴를 제한하고, 후퇴가 길어지면 측면 재정렬과 밀어내기로 전환
- 포기 버튼 추가로 진행 중인 런을 즉시 종료 가능
- 초기 중앙 전투 버튼이 새 런을 다시 만들며 스탯을 초기화하던 문제 수정
- 창·방어형 미러전에서 뒤로만 거리두는 루프 완화
- 원거리에서 밀치기·무기 충돌로 자세가 흐트러지던 오판정 완화
- 단검 미러전에서 붙어서 빙글 도는 루프를 짧은 교전으로 전환
- 무기 충돌과 공격 성공 시 상대의 몸 방향이 즉각 흐트러지도록 반영
- 캐릭터 세팅, 중앙 전투·결과·보상, 우측 탑 정보·상태 UI 구조 적용
- 클릭 가능한 UI는 매 프레임 재생성하지 않고 상태 변화 시에만 갱신

- 패링 기본형, 무기별 패링 효율, 반격 창 추가
- 무기별 공격 준비·타격·후딜 리듬 강화
- 명중, 패링, 무기 충돌 시 짧은 충격 정지로 타격 순간 강조
- 공격 상태에 따라 무기 궤적과 타격 범위가 더 분명하게 표시
- 패배하면 새 런 시작

```text
│  └─ utils.js
├─ README.md
├─ CHANGELOG.md

├─ .gitignore
└─ .nojekyll
```

## 파일별 역할

- `index.html`: 화면 구조
- `css/style.css`: 디자인
- `js/data.js`: 무기, 성격, 스탯, 스킬, 보상, 층 성장 규칙
- `js/state.js`: 런 생성, 층 이동, 유닛 생성, 랜덤 상대 생성, 보상 적용
- `js/battle.js`: 공격 판정, 피해 계산, 승패 판정, 사각형 경계 처리
- `js/ai.js`: 1:1 자동 이동 판단
- `js/render.js`: 캔버스 그리기
- `js/main.js`: UI 연결과 단일 게임 루프
- `js/utils.js`: 공통 수학 함수

## 로컬 실행

`type="module"`을 사용하므로 파일을 더블클릭하면 브라우저 보안 정책에 막힐 수 있습니다.
아래처럼 로컬 서버로 실행하세요.

```bash
python -m http.server 8000
```

브라우저에서 접속합니다.

```text
http://localhost:8000
```

```text
Source: Deploy from a branch
Branch: main
Folder: /root
```

## 개발 원칙

- 기존 파일 아래에 패치 블록을 추가하지 않습니다.
- 같은 이름의 함수를 다시 선언하지 않습니다.
- 수정은 담당 파일의 기존 함수를 직접 수정합니다.
- `requestAnimationFrame`은 `main.js`에서만 호출합니다.
- 전투 계산은 `battle.js`, 화면 그리기는 `render.js`에서만 처리합니다.



## v0.4.6 핵심 변경

- 방어형이 낮은 체력에서 뒤로만 도망가는 루프를 줄였습니다.
- 창이 근접한 상대를 상대로 계속 뒤로만 빠지는 현상을 줄이고, 측면 재정렬과 밀어내기로 전환하도록 했습니다.
- 후퇴가 일정 시간 이상 이어지면 후퇴 제한 상태가 걸리고, 측면 이동 또는 견제 행동으로 바뀝니다.
- 창 최소거리 안쪽 또는 방어형이 붙잡힌 상태에서는 짧은 밀어내기 행동으로 상대를 떼어냅니다.
- 무기 충돌이 발생하면 양쪽 자세 게이지가 깎이고, 몸 방향이 즉각 틀어지며 짧은 후딜이 생깁니다.
- 공격 성공 시에도 상대의 몸 방향이 즉각 흐트러지도록 `battle.js`의 기존 피격 흐름을 직접 수정했습니다.
- 자세가 0이 되면 스태미너 붕괴처럼 잠깐 굳고, 이동·회전·공격이 크게 제한됩니다.
- 모든 수정은 기존 담당 파일의 기존 함수와 데이터 구조를 직접 수정하는 방식으로 반영했습니다.

## v0.4.4 핵심 변경

- 전체 화면을 캐릭터 세팅 영역, 중앙 전투 영역, 우측 탑 정보·상태 영역으로 재배치했습니다.
- 승리 결과와 보상 선택을 중앙 전투 공간의 오버레이에서 처리하도록 정리했습니다.
- 스탯 버튼이 매 프레임 다시 생성되어 클릭이 무시되던 문제를 수정했습니다.
- 스탯 버튼과 보상 버튼은 이벤트 위임과 렌더링 키를 사용해 상태가 바뀔 때만 갱신합니다.
- 전투 중에는 무기·성격 선택을 잠그고, 전투 전과 다음 층 대기 상태에서만 스탯을 배분할 수 있습니다.
- 모든 수정은 기존 담당 파일의 기존 구조를 직접 수정하는 방식으로 반영했습니다.

## v0.4.3 핵심 변경

- 무기별 방향 전환 속도를 적용해 상대가 즉시 돌아보지 못하게 했습니다.
- 공격 준비, 타격 판정, 후딜 상태에서는 회전 속도가 제한됩니다.
- 공격 시작 시 상대를 즉시 바라보던 처리를 제거해 공격 실패와 빈틈이 생기도록 했습니다.
- 자세 게이지를 추가했고, 자세가 0이 되면 짧은 자세 흐트러짐 상태가 발생합니다.
- 단검은 상대의 후딜과 자세 흐트러짐을 측후방 진입 기회로 활용합니다.
- 상태 패널과 캔버스에 자세 게이지를 표시합니다.
- 모든 수정은 기존 담당 파일의 기존 로직을 직접 수정하는 방식으로 반영했습니다.

## v0.4.2 핵심 변경

- 새 런 시작 시 기본 스탯 포인트 10점을 제공합니다.
- 보상 선택 후 다음 층으로 이동할 때 플레이어 체력이 완전히 회복됩니다.
- 다음 층으로 이동한 뒤 바로 전투가 시작되지 않고, 스탯 포인트를 배분한 뒤 전투를 시작할 수 있도록 흐름을 정리했습니다.
- 전투 시작 전 스탯 배분 시 상대가 다시 랜덤 생성되지 않고 플레이어 상태만 갱신됩니다.
- 모든 수정은 기존 담당 파일의 로직을 직접 수정하는 방식으로 반영했습니다.


## v0.4.0 핵심 변경

## v0.4.1 핵심 변경
