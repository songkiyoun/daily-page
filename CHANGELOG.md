# CHANGELOG

## v0.5.2
- 무기별 공격 진입 동작을 더 뚜렷하게 분리
- 창은 준비 중 짧게 잡고 타격 순간 앞으로 훅 들어오며 길게 찌르는 느낌 강화
- 서양검은 부채꼴 범위를 조금 줄이고 묵직한 전진 베기 강화
- 동양검은 빠른 사선 진입과 짧은 회수 리듬 강화
- 단검은 페이크성 측면 흔들림과 짧은 침투, 빠른 이탈 강화
- 공격 방향은 공격 시작 시점의 각도를 더 유지하도록 정리해 헛침과 빈틈이 생기도록 수정
- 기존 함수와 데이터 구조를 직접 수정했으며 새 패치 블록 추가 없음

## v0.5.1
- 무기별 공격 준비, 타격, 후딜 리듬 강화
- 창, 서양검, 동양검, 단검의 activeFrames, 헛침 후딜, 충격 정지, 시각 스윙 수치 추가
- 공격 명중, 패링, 무기 충돌 시 짧은 impact stop 적용
- 헛친 공격은 무기별 missRecoveryAdd만큼 후딜이 늘어나도록 수정
- 캔버스의 무기 궤적, 타격 범위, 무기 선 길이를 공격 상태에 맞게 표시
- 기존 함수 직접 수정 방식 유지, 새 패치 블록 추가 없음

## v0.4.9
- 패링 기본형 추가
- 무기별 패링 효율과 성격별 패링 보정 적용
- 정면에서 들어오는 공격만 패링 가능하도록 전방 각도와 거리 조건 정리
- 패링 성공 시 공격자 자세 피해, 몸 방향 틀어짐, 짧은 후딜, 밀림 발생
- 패링 성공자에게 짧은 반격 창을 부여해 다음 공격 피해와 자세 피해가 소폭 증가하도록 수정
- 패링 성공 시 캔버스에 짧은 푸른 링 표시
- 패치 블록을 추가하지 않고 기존 `data.js`, `state.js`, `battle.js`, `render.js` 구조를 직접 수정

## v0.4.8
- 중앙 전투 오버레이의 시작 버튼이 새 런을 다시 생성해 배분한 스탯을 초기화하던 문제를 `main.js`의 기존 시작 흐름 직접 수정으로 해결
- 포기 버튼을 추가해 진행 중인 런을 즉시 패배 처리하고 새 캐릭터 생성으로 돌아갈 수 있도록 정리
- 창·방어형, 특히 창 방어형 미러전에서 뒤로만 거리두는 루프를 줄이도록 `ai.js`의 후퇴·견제 판단을 직접 수정
- 창 미러전은 거리 유지 후 후퇴가 아니라 측면 견제와 짧은 재진입을 우선하도록 변경
- 밀치기와 방어형 견제 밀어내기가 실제 근접 또는 최소거리 안쪽에서만 발생하도록 `battle.js`의 조건을 좁혀 원거리 자세 오판정을 완화
- 무기 충돌은 양쪽이 실제 타격 판정 상태일 때만 발생하도록 조정해 떨어져 있는데 자세가 흐트러지는 현상을 줄임
- 단검 측후방 진입에 순간 가속을 추가하고, 단검 미러전은 빙글빙글 도는 루프 대신 짧은 교전·이탈·재진입 흐름으로 수정
- 패치 블록을 추가하지 않고 기존 함수와 데이터 구조를 직접 수정

## v0.4.6
- 방어형이 일정 체력 이하에서 뒤로만 도망가는 루프를 줄이도록 `ai.js`의 기존 후퇴 판단을 직접 수정
- 창이 가까이 붙은 상대를 상대로 계속 후퇴만 하지 않고 측면 재정렬과 밀어내기로 전환하도록 변경
- 후퇴가 일정 시간 이상 이어지면 후퇴 제한 상태를 걸어 측면 이동 또는 견제 행동을 우선하도록 정리
- 창 최소거리 안쪽 또는 방어형이 붙잡힌 상태에서 짧은 밀어내기 행동을 수행하도록 `battle.js`의 기존 이동·전투 흐름 직접 수정
- 공격 성공 시 상대의 몸 방향이 즉각 틀어지도록 자세 흐트러짐을 방향 변화까지 포함하는 구조로 변경
- 무기 충돌 판정을 추가해 양쪽 자세 피해, 방향 틀어짐, 짧은 후딜이 발생하도록 정리
- 자세가 0이 되면 스태미너 붕괴처럼 잠깐 굳고 이동·회전·공격이 크게 제한되도록 수정
- 패치 블록을 추가하지 않고 기존 함수와 데이터 구조를 직접 수정
